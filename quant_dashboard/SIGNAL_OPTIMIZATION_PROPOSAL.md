## 一、当前系统评估

### 1.1 已完成的信号框架

信号评分系统已实现机构级的七维度框架：

| 因子 | 满分 | 权重 | 核心逻辑 |
|:-----|:----|:-----|:--------|
| ① RSI 超卖深度 | 25 | 25% | 基于 Regime-adaptive RSI 阈值分档打分 |
| ② RSI 动量方向 | 15 | 15% | ΔRSI₅ ≥ +3 → 15分，捕捉底部反转动量 |
| ③ BIAS 乖离率 | 20 | 20% | 基于 Regime-adaptive BIAS 阈值 |
| ④ 布林带 %B | 15 | 15% | %B ≤ 0.10 → 15分，捕捉下轨超卖 |
| ⑤ 成交量信号 | 10 | 10% | U型评分：≥2.0x 恐慌放量或 ≤0.5x 绝望缩量 |
| ⑥ 趋势位置 | 10 | 10% | 价格相对 MA 的位置 |
| ⑦ K线形态 | 5 | 5% | 锤子线/下影线/十字星 |

入场门槛按 Regime 自适应：BULL 60 / RANGE 68 / BEAR 78 / CRASH 999。

### 1.2 当前的核心问题：一套参数覆盖全部资产

```python
# 当前 RPM 参数 — 所有资产类型共享同一套参数
FALLBACK_PARAMS = {
    "BEAR":  {"rsi_buy": 45, "bias_buy": -3.0, "stop_loss": 0.05},
    "RANGE": {"rsi_buy": 40, "bias_buy": -2.0, "stop_loss": 0.07},
    "BULL":  {"rsi_buy": 45, "bias_buy": -1.5, "stop_loss": 0.06},
}
```

**问题**: 沪深300ETF (年化波动 ~18%) 和科创芯片ETF (年化波动 ~35%) 使用**完全相同的** RSI 买入阈值、BIAS 乖离阈值和止损幅度。这违反风险管理的基本原则——波动率越高，止损应越宽，入场应越谨慎。

### 1.3 四类资产的风险特征差异

| 维度 | A股个股 | 行业ETF | 宽基ETF | 海外宽基ETF |
|:-----|:------|:------|:------|:----------|
| **年化波动率** | 35-50% | 25-40% | 18-25% | 15-22% |
| **单日极端波动** | ±10% (涨跌停) | ±5-8% | ±3-5% | ±2-4% (无涨跌停) |
| **流动性风险** | 高 (小盘股) | 中 | 低 | 低 |

| **适合的止损模型** | 宽止损+快止损 | 中等 | 窄止损+慢止损 | 汇率调整止损 |

## 二、优化方案：四类资产差异化参数矩阵

### 2.1 总体设计原则

1. **波动率锚定**: 所有阈值基于资产类别的年化波动率自动缩放
2. **流动性缓冲**: 低流动性资产额外增加安全边际
3. **回测校准**: 所有参数基于 2021-2025 五年回测数据验证
4. **Regime × Asset 交叉**: 牛市放宽/熊市收紧的 Regime 调节叠加资产类别基础参数

### 2.2 核心公式

```
adjusted_threshold = base_threshold × σ_asset / σ_baseline × regime_multiplier
```

其中:
- `base_threshold`: 资产类别的基础阈值
- `σ_asset`: 该资产的年化波动率（从日线计算）
- `σ_baseline`: 基准波动率（沪深300 = 20%）
- `regime_multiplier`: Regime 调节系数 (BULL=1.2, RANGE=1.0, BEAR=0.8, CRASH=0.6)

## 三、四类资产差异化规则详表

### 3.1 A股个股 (Individual Stocks)

**风险特征**: 高波动、高尾部风险、易受事件驱动、流动性不均。
**策略定位**: 辅助验证工具（副仓 ≤20%），仅在 ETF 信号共振时入场。

| RSI 买入阈值 (RANGE) | 40 | **35** | 震荡市个股波动大，收窄入场条件防假信号 |
| RSI 买入阈值 (BULL) | 45 | **40** | 牛市中个股回调也可能很深，保持谨慎 |
| BIAS 乖离买入 (BEAR) | -3.0% | **-5.0%** | 个股单日可跌 10%，-3% 太容易触发 |
| BIAS 乖离买入 (RANGE) | -2.0% | **-4.0%** | |
| BIAS 乖离买入 (BULL) | -1.5% | **-3.0%** | |
| 硬止损 | -10% | **-12%** | 个股波动大，-10% 容易被震出。但止损后需等 20 日冷却 |
| T1 止盈 | +15% | **+20%** | 个股爆发力强，提高止盈目标 |
| 入场门槛 (BULL) | 60 | **65** | 个股风险高，即使牛市也需更严 |

### 3.4 海外宽基 ETF (Overseas ETFs)

**风险特征**: 汇率风险叠加、隔夜跳空风险、流动性一般（A股市场交易的跨境ETF）。
**策略定位**: 全球配置辅助（建议 ≤30% 总仓位）。

| 参数 | 当前值 (通用) | 优化后 | 理由 |
|:-----|:----------|:-----|:-----|

## 四、止损规则分级优化

### 4.1 当前止损体系

```
硬止损 (HARD STOP):  ETF -8% / 个股 -10%
时间止损 (TIME STOP): 20日无进展
市场熔断 (CIRCUIT):   CSI300 单周 -10%
```

### 4.2 优化后的四级止损体系

| 级别 | 触发条件 | A股个股 | 行业ETF | 宽基ETF | 海外ETF | 操作 |
|:----|:--------|:------|:------|:------|:------|:----|
| **L1 追踪止盈** | 浮盈 > T1 目标 | 最高点回撤 -12% | 回撤 -8% | 回撤 -5% | 回撤 -6% | 清出 50% |
| **L2 硬止损** | 浮亏达阈值 | -12% | -8% | -6% | -8% | 全部清仓 |
| **L3 时间止损** | 持仓超 N 日且收益 < +5% | 15 日 | 20 日 | 30 日 | 25 日 | 减半仓 |
| **L4 宏观熔断** | 系统性风险 | CSI300 -10%/周 或 VIX>35 或 AIAE R5 | | | | 全部清仓 |

### 4.3 新增：动态追踪止盈 (ATR-based)

当前追踪止盈是固定百分比（最高点回撤 -8%）。优化为基于 ATR 的动态追踪：

```
trailing_stop = max_price × (1 - ATR_multiplier × ATR(14) / max_price)
```

| 资产类别 | ATR 倍数 | 效果 |
|:---------|:-------|:-----|
| 个股 | 3.0x | 宽止损，容忍噪声 |
| 行业 ETF | 2.5x | 中等宽度 |
| 宽基 ETF | 2.0x | 窄止损，快速锁利 |
| 海外 ETF | 2.5x | 中等宽度 |

## 五、仓位管理的资产差异化

### 5.1 当前问题

当前 `max_pos` 是硬编码在 `MR_UNIVERSE` 列表中的静态值（如 `512480.SH: 8%`, `510300.SH: 15%`）。没有机制根据波动率和 Regime 动态调整。

### 5.2 优化方案

```python
def dynamic_position_cap(asset_class, annual_vol, regime):
    base_cap = {
        "individual_stock": 8,
        "sector_etf": 12,
        "broad_etf": 20,
        "overseas_etf": 15,
    }[asset_class]

    # 波动率缩放: 高波动 → 降低仓位
    vol_scale = 20 / max(annual_vol, 10)  # 基准波动 20%
    vol_scale = max(0.5, min(vol_scale, 1.5))

    # Regime 缩放
    regime_scale = {"BULL": 1.0, "RANGE": 0.85, "BEAR": 0.6, "CRASH": 0.0}[regime]

    return round(base_cap * vol_scale * regime_scale, 1)
```

### 5.3 示例

| 标的 | 资产类别 | 年化波动 | Regime | 当前 max_pos | 优化后 max_pos |
|:-----|:--------|:-------|:------|:----------|:-----------|
| 科创芯片ETF (588200) | 行业 ETF | 35% | RANGE | 7% | 12×0.57×0.85 = **5.8%** |
| 沪深300ETF (510300) | 宽基 ETF | 18% | BULL | 15% | 20×1.11×1.0 = **22% → 20%** |
| 标普500ETF (513500) | 海外 ETF | 16% | RANGE | 无明确 | 15×1.25×0.85 = **15.9%** |

> **预案总结**: 当前信号评分系统的基础框架（7维100分制 + Regime自适应）是机构级的。核心短板是"一套参数打天下"——科创板芯片ETF和沪深300ETF使用完全相同的RSI阈值和止损线，这在专业量化中是原则性缺陷。本预案的核心改动量不大（主要是在参数层增加资产类别维度），但能系统性提升信号质量——减少高波动资产的假信号，让低波动资产的趋势利润吃满。

> **优先建议**: Phase 1（参数矩阵）应优先执行——改动集中在 `mr_per_regime_params.json` 和引擎参数读取逻辑，不影响现有数据流，风险最低，收益最高。
